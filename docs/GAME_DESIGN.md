# MindMine — キーボード地雷ゲーム

キーボードのキーを「長押し→離す」で地雷を設置し、再度同じキーを押して起爆するアクションゲームです。
チャージ時間で爆破範囲が決まり、敵を倒してスコアを稼ぎます。

---

## ファイル構成

| ファイル | 役割 |
|---|---|
| `index.html` | HTML 構造・UI 要素の定義 |
| `style.css` | 全 UI スタイル |
| `game.js` | ゲームロジック全体（約 1,400 行） |

`game.js` の冒頭に定数ブロックがあり、ゲームの全パラメータはここに集約されています。
PixiJS v7（WebGL）でグラフィックを描画し、Web Audio API でサウンドを合成しています（外部アセット不要）。

---

## 操作方法

| 操作 | 内容 |
|---|---|
| **任意のキーを長押し → 離す** | そのキーに対応した地雷をカーソル位置に設置 |
| **設置済みキーを再度押す** | 地雷を起爆（設置から 3 秒後以降のみ） |

追跡するキーは `game.js` の `TRACK_KEYS` で定義されています。

```js
const TRACK_KEYS = 'ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789-^\\@[;:],./'.split('');
```

## 地雷の仕様

### チャージと爆破半径

キーを押している時間（チャージ時間）が爆破半径を決定します。

```
爆破半径 = BLAST_MIN + (チャージ時間 / CHARGE_FULL_MS) × (blastMax - BLAST_MIN)
```

| 定数 | 値 | 意味 |
|---|---|---|
| `CHARGE_FULL_MS` | 2000 ms | この時間押し続けると最大チャージ |
| `BLAST_MIN` | 40 px | 即離ししたときの最小爆破半径 |
| `blastMax` | `min(gameW, gameH) / 4` | 最大爆破半径（ゲーム開始時のウィンドウサイズから算出） |

チャージ中はカーソル周囲に点線円でプレビューが表示されます（`drawChargePreview` 関数）。

### 不発弾

`CHARGE_FULL_MS` を超えてキーを押し続けてから離すと、**不発弾**になります。

- 不発弾は通常起爆できません（同じキーを押すと地雷が震えてエラー音）
- **連鎖爆発（誘爆）でのみ**破壊できます
- 誘爆時の爆破半径は通常と同じ計算値が使われます（青い爆発色）

不発弾かどうかは設置時に確定します（`game.js` keyup ハンドラ）：

```js
const isDud   = held > CHARGE_FULL_MS;
const clamped = Math.min(held, CHARGE_FULL_MS);  // 超過分は切り捨て
const br      = BLAST_MIN + (clamped / CHARGE_FULL_MS) * (blastMax - BLAST_MIN);
```

### 起爆待機時間

設置直後は起爆できません。設置から `MINE_READY_MS`（3000 ms）が経過して初めて起爆可能になります。
これはチャージ時間（`CHARGE_FULL_MS`）とは独立した別の値です。

```js
const MINE_READY_MS = 3000;  // 設置→起爆可能になるまでの待機時間 (ms)
```

### 画面サイズと活動範囲

`gameW` / `gameH` および `blastMax` はゲーム**開始時**のウィンドウサイズで固定され、
プレイ中のリサイズは無視されます（`initState()` で設定）。
敵の活動範囲もこのサイズを基準にしています。

## 敵の仕様

### 出現と進化

敵は画面外から一定間隔でスポーンし、6 段階（ステージ 0〜5）に進化します。

| 定数 | 値 | 意味 |
|---|---|---|
| `BASE_SPAWN_MS` | 2800 ms | Wave 1 のスポーン間隔 |
| `WAVE_INTERVAL` | 20000 ms | Wave が上がるまでの時間（20 秒） |
| `EVO_MS_MIN` | 20000 ms | 最大サイズ敵の進化間隔（20 秒） |
| `EVO_MS_MAX` | 30000 ms | 最小サイズ敵の進化間隔（30 秒） |
| `MAX_STAGE` | 5 | 最終ステージのインデックス（0 始まりで 6 段階） |

敵のサイズ（半径）は `ENEMY_R_MIN`（14 px）〜`ENEMY_R_MAX`（26 px）でランダムに決まります。
**大きい敵ほど速く進化**します（`spawnEnemy` 関数）：

```js
const rNorm   = (r - ENEMY_R_MIN) / (ENEMY_R_MAX - ENEMY_R_MIN);
const baseEvo = EVO_MS_MAX - rNorm * (EVO_MS_MAX - EVO_MS_MIN);  // 30000〜20000 ms
```

### 移動パターン

スポーン時にランダムで 5 種類の移動パターンが割り当てられます（`MOVE_TYPES` 配列）：

| パターン | 内容 |
|---|---|
| 0 | 直進 |
| 1 | サイン波（蛇行） |
| 2 | 緩やかなカーブ |
| 3 | ジグザグ |
| 4 | 弧を描く旋回 |

### ステージ外観

敵の各ステージは `STAGE_STYLE` 配列で色と大きさが定義されています：

| ステージ | 色 | 特徴 |
|---|---|---|
| 0 | 水色 | 基本サイズ |
| 1 | 草緑 | やや大きい |
| 2 | 黄色 | |
| 3 | オレンジ | |
| 4 | 赤 | 赤黒点滅で警告表示 |
| 5 | 暗黒赤 | 最終段階 |

ステージ 4（最終段階の一つ手前）は赤黒で点滅して危険を警告します。
最終ステージ（5）の敵が出現してから `FINAL_HORROR_MS`（5000 ms）以内に倒せないとゲームオーバーになります。

## スコア計算

1 回の爆発で敵を倒したとき、以下の順で計算されます（`detonateAt` 関数）：

```
基礎点 = 倒した敵ごとに (10 + ステージ × 5) の合計
　　　　 ジャストミート判定なら ×2
基礎点 += 現在のコンボ数（1 コンボにつき +1 pt）
コンボ数 += 今回倒した数
```

```
爆破範囲補正 = 1 + (blastMax - 爆破半径) / (blastMax - BLAST_MIN)
　　　　　　  ※ 爆破範囲が最小のとき最大 2 倍、最大のとき 1 倍
最終スコア = round(基礎点 × 爆破範囲補正) × ランク倍率
```

| ルール | 内容 |
|---|---|
| 敵の基礎点 | ステージ 0 = 10pt、ステージ 5 = 35pt |
| ジャストミート | 地雷本体（半径 10 px）と敵が重なっていた場合に基礎点 ×2 |
| 爆破範囲補正 | 狭いほど高倍率（最大 2 倍）|
| コンボ | 敵を連続して倒すと累積。外れると 0 リセット |
| ランク倍率 | `ランク番号` 倍（ランク 1 = ×1、ランク 10 = ×10） |

## プレイヤーランク

### ランクアップ / ダウン判定

`RANK_INTERVAL_MS`（20 秒）ごとに、その間に**出現した敵の数**と**倒した数**を比較して評価します。

```js
const RANK_INTERVAL_MS    = 20000;
const RANK_UP_THRESHOLD   = 0.6;   // 出現数の 60% 以上倒す → ランクアップ
const RANK_DOWN_THRESHOLD = 0.4;   // 出現数の 40% 以下しか倒せない → ランクダウン
```

| 撃破率 | 結果 |
|---|---|
| 60% 以上 | ランク +1 |
| 40% 以下 | ランク -1（最低はランク 1） |
| その間 | 変化なし |

ランクは上限なしで無限に上昇します（`getRankLabel` / `getRankMult` 関数）。

### ランクの影響

ランクが上がるほど難しく、かつ高得点になります：

| 要素 | 変化 |
|---|---|
| スコア倍率 | ランク番号 × 1 倍（ランク 3 なら ×3） |
| 敵スポーン間隔 | ランク 1 段階ごとに 10% 短縮（最短 500 ms） |
| 敵の進化速度 | ランク 5 段階までは 10%/段階 短縮、以降は固定（最短 10 秒） |

スポーン間隔の計算（`game.js` メインループ）：

```js
const baseInterval  = Math.max(800, BASE_SPAWN_MS - (wave - 1) * 200);
const rankSpeedMult = 1 + rankIndex * 0.1;
const spawnInterval = Math.max(500, baseInterval / rankSpeedMult);
```

### リアルタイム表示

画面左上にランクメーターが常時表示されます。20 秒の判定期間内の撃破率をバーとパーセントで確認できます。

## ゲームオーバー条件

最終ステージ（ステージ 5）の敵が出現した瞬間からカウントが始まり、
`FINAL_HORROR_MS`（5000 ms = 5 秒）以内に倒せなければゲームオーバーになります。

ゲームオーバー画面には以下が表示されます：
- 最終スコア
- 最終コンボ数
- 最終ランク
- 撃破速度（匹/分）

## Wave システム

`WAVE_INTERVAL`（20 秒）ごとに Wave が 1 上昇し、敵のスポーン間隔が短くなります：

```js
const baseInterval = Math.max(800, BASE_SPAWN_MS - (wave - 1) * 200);
// Wave 1: 2800ms, Wave 2: 2600ms, ... Wave 11 以降: 800ms（下限）
```
